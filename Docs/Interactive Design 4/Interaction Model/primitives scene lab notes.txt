Yes — there are genuinely different configurations inside the bound category, beyond just tweaking curve/smoothing/range.

What you’re feeling is correct:
there are qualitatively different behaviors, not just parameter variations.

The key is this:

Bound is one relationship type, but it has distinct interaction modes.

1. The real decomposition of “bound”

Within bound, you actually have 3 fundamentally different configurations:

(A) Direct Tracking

Output follows input continuously

classic mapping
“the system follows you”

→ what you already planned (C1–C3)

(B) Thresholded / Gated Tracking

Tracking only happens in certain regions

below threshold → nothing
above threshold → tracking

→ still bound, but conditional

(C) Accumulative / Integrative Tracking

Output depends on history of input, not just current value

input drives accumulation over time
still continuously coupled

→ proto-state, but still feels “bound”

👉 These are not just parameter changes
They change the structure of the relationship
