# Multimedia Design 4 — Interaction Design for Artists

## Course Identity

- **Official module**: Multimedia Design 4
- **Actual focus**: Interaction design as expressive/authorial medium
- **Core thesis**: Agency as art — interaction itself is the creative material, not just a technical layer
- **Target students**: Artists, architects, designers (not CS students)
- **Philosophical position**: The art/installation world defaults to static, "look don't touch" experiences. This course challenges that by treating participant agency as the primary expressive tool.

## Hard Constraints

- **Total hours**: 40
- **Session format**: 2.5 hours per lecture
- **Number of sessions**: 16
- **Tool**: Unity
- **Students' starting point**: No coding background; some 3D and environmental design experience
- **Teaching language**: Techniques and tools, not first-principles C#

## Course Objectives

1. **Acquire the perspective of interactive art** — understand agency as an expressive medium; learn to see, analyze, and critique interaction as the primary material of the work
2. **Acquire technical implementation knowledge** — learn a specific set of interaction techniques and production practices through practical examples in Unity
3. **Produce a personal project** — create an interactive work that demonstrates both the perspective and the technical ability

## Project & Production Practice

- Project work starts **from session one** — not after "learning the tools"
- Part of each session is dedicated to production practices and individual project follow-up
- Students receive direct mentorship: direction, constraints, redirection
- Core production values:
  - **Don't be precious** — don't cling to the first idea
  - **Experiment broadly** — produce 10 things to find the 1 that works
  - **Test relentlessly** — if it hasn't been tested, it doesn't exist
  - **Navigate constraints** — find the right scope, the right limits, the right focus
  - **Explore before committing** — the project will change; that's the process

## Pedagogical Approach

- **Project-based**: Each session introduces a small, complete interactive project as teaching material
- **Structure per session**: Brief theory/reference → technique breakdown → hands-on making
- **Real-world anchoring**: Each project draws from a real interactive artwork or installation as reference/inspiration
- **Progression**: Simple dynamic environments → layered, meaningful interactions
- **Final output**: Students produce a personal interactive project

## Core Philosophy

- Agency is the medium — actions function as metaphors, not just mechanics
- The participant becomes co-author of the experience
- Interaction design = choosing what the audience *does*, not just what they *see*
- The aesthetic goal: **epiphany through action** — meaning shining through doing, not layered on top (see [why-agency-art.md](why-agency-art.md))
- Key reference: C. Thi Nguyen, *Games: Agency As Art* (2020) — see [nguyen-agency-as-art.md](nguyen-agency-as-art.md) for full breakdown
  - Games are an art form whose medium is agency itself
  - Layered agency: designer sculpts the possibility space, player navigates it
  - Striving play: the texture of deciding and acting has its own aesthetic value
  - Agency has designable qualities — fluid, constrained, agonizing, liberating
  - Crystallization: games concentrate rare aesthetic experiences of action
  - Process art: the beauty is in the actions produced, not the artifact itself
  - Value clarity vs. value capture: crisp goals are powerful but dangerous

## Interaction Areas (to be scoped and ordered)

1. **Core object manipulation** — transforms, spawning, resetting state
2. **UI-based interaction** — menus, simulated inputs, ubiquitous/cross-platform input design
3. **Character control** — first-person movement, walkability
4. **Physics interactions** — forces, collisions, constraints
5. **NPC / narrative interaction** — characters, dialogue, behavioral responses
6. **Environment manipulation** — light, weather, ambiance, world-state changes
7. **Sensory feedback** — sound cues, visual effects, haptics tied to player actions
